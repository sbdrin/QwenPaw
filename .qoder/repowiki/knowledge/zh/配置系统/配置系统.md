# 配置系统

QwenPaw 应用平台采用分层配置系统，结合了文件配置、环境变量和运行时上下文变量，为后端服务和前端控制台提供了灵活的配置管理。

## 核心配置方法

### 1. 文件配置
- **主配置文件**: `config.json` 存储在工作目录中（默认 `~/.qwenpaw/config.json`）
- **环境变量存储**: `envs.json` 存储在安全目录中（默认 `~/.qwenpaw.secret/envs.json`），值经过加密
- **配置模型**: 使用 Pydantic 模型定义配置结构，支持类型验证和默认值设置

### 2. 环境变量处理
- **EnvVarLoader 类**: 提供类型安全的环境变量读取方法（`get_bool`, `get_int`, `get_float`, `get_str`）
- **向后兼容**: 支持从 `QWENPAW_*` 到 `COPAW_*` 的自动回退查找
- **加密存储**: 敏感环境变量在 `envs.json` 中加密存储

### 3. 运行时上下文变量
- **ContextVars**: 使用 Python 的 `contextvars` 模块传递运行时特定信息（如当前工作目录、会话ID、工具包实例等）
- **作用域隔离**: 在多代理环境中确保配置的正确传递

## 配置架构

### 后端配置
- **主配置**: `src/qwenpaw/config/config.py` 定义了所有配置模型，包括通道配置、心跳配置、内存配置等
- **配置加载**: `src/qwenpaw/config/utils.py` 实现了带缓存的配置加载和验证逻辑
- **环境变量**: `src/qwenpaw/envs/store.py` 处理环境变量的持久化和加载

### 前端配置
- **API URL**: 通过 `VITE_API_BASE_URL` 环境变量确定后端API地址
- **认证令牌**: 支持本地存储的令牌和构建时注入的令牌

## 开发者规则

1. **配置模型定义**:
   - 使用 Pydantic BaseModel 定义配置结构
   - 为敏感数据添加适当的验证规则
   - 使用 `Field` 注解指定默认值和约束

2. **环境变量访问**:
   - 使用 `EnvVarLoader` 类的方法获取环境变量
   - 不要直接访问 `os.environ`
   - 对于敏感信息，使用 `envs.json` 加密存储

3. **运行时上下文**:
   - 在需要跨函数传递上下文信息时使用 ContextVar
   - 确保在适当的作用域内设置和获取上下文变量

4. **配置文件管理**:
   - 主配置文件应保持向后兼容性
   - 使用配置验证防止无效配置导致的应用崩溃
   - 实现配置变更的热重载机制

5. **安全性考虑**:
   - 敏感配置信息必须加密存储
   - 避免在日志中输出敏感配置值
   - 使用权限控制限制对配置文件的访问

该配置系统确保了应用在不同环境下的灵活性，同时保护了敏感配置信息的安全性。通过分层设计，开发者可以轻松地覆盖默认配置而无需修改代码。此外，配置系统还支持插件通道的动态配置，允许第三方插件扩展配置选项。
