# 前端样式系统

QwenPaw 桌面控制台采用基于 Ant Design 的样式系统，结合自定义主题和深色模式支持。

## 核心技术栈

- **Ant Design**: 使用 antd 作为主要的 UI 组件库
- **@agentscope-ai/design**: AgentScope 设计系统，在 antd 基础上进行定制
- **Less**: CSS 预处理器，用于编写样式
- **antd-style**: 提供全局样式和主题管理能力
- **CSS Modules**: 在部分组件中使用模块化样式

## 主要样式文件

- `console/src/styles/layout.css`: 主要布局和深色模式样式，包含大量针对 antd 组件的深色模式覆盖
- `console/src/styles/form-override.css`: 表单相关样式保护，防止第三方组件样式覆盖
- 各组件的 `.module.less` 文件：局部样式模块

## 深色模式实现

深色模式通过在 `<html>` 元素上添加 `dark-mode` 类来实现。`layout.css` 中包含了大量针对该类的样式覆盖，为所有 antd 组件提供深色变体。主题切换由 `ThemeContext` 管理，支持浅色、深色和跟随系统三种模式。

## 样式架构

1. **全局样式**: 通过 `layout.css` 和 `form-override.css` 定义
2. **组件样式**: 使用 antd 组件库和 @agentscope-ai/design 组件库
3. **主题管理**: 通过 antd 的主题系统和自定义 CSS 变量实现
4. **响应式设计**: 在样式中考虑了移动端适配（如安全区域）

## 开发规范

- 使用 Less 编写样式
- 组件级样式优先使用 CSS Modules（.module.less）
- 全局样式修改应集中在 `layout.css` 中
- 深色模式样式必须与浅色模式保持一致的结构，仅改变颜色值
- 遵循 antd 的组件使用规范
- 对于第三方组件库的样式覆盖，应使用明确的选择器并添加注释说明
