# 依赖管理策略

QwenPaw 全栈应用平台采用多语言、多包管理器的依赖管理策略，分别处理前端、后端和桌面应用的依赖。

## 前端依赖管理（JavaScript/TypeScript）

前端控制台位于 `console/` 目录，使用 npm 作为包管理器。主要特点：

- 使用 `package.json` 定义依赖项，包括 React 生态系统、Ant Design 组件库、Tauri API 等
- 依赖版本采用语义化版本控制，如 `^` 和 `~` 符号
- 包含特定版本的内部库，如 `@agentscope-ai/chat` 版本为 `1.1.72-beta.1783494781362`
- 使用 `package-lock.json` 锁定依赖树以确保构建一致性

## 后端依赖管理（Python）

后端核心位于 `src/` 目录，使用 Python 的标准依赖管理机制。主要特点：

- 通过 `pyproject.toml` 文件定义项目元数据和依赖项
- 使用 `setup.py` 或 setuptools 进行包构建
- 依赖列表包含多个 AI 相关库，如 `agentscope==2.0.4`、`transformers`、`openai` 等
- 定义了多个可选依赖组，如 `test`、`dev`、`local`、`whisper`、`sip` 等
- `src/qwenpaw.egg-info/requires.txt` 文件包含了实际解析后的依赖列表

## 桌面应用依赖管理（Rust）

Tauri 桌面应用位于 `console/src-tauri/` 目录，使用 Cargo 作为 Rust 包管理器。主要特点：

- 通过 `Cargo.toml` 文件定义 Rust 依赖项
- 包含 Tauri 框架及其相关插件，如 `tauri`、`tauri-plugin-log`、`tauri-plugin-shell` 等
- 使用 `Cargo.lock` 文件锁定具体的依赖版本

## 依赖管理规则

开发者应遵循以下规则来管理依赖：

1. 对于前端依赖，使用 `npm install` 添加新包，并确保 `package-lock.json` 更新
2. 对于 Python 依赖，更新 `pyproject.toml` 中的依赖列表，并重新运行包安装命令
3. 对于 Rust 依赖，更新 `Cargo.toml` 并让 Cargo 自动处理依赖解析
4. 在添加新依赖时，考虑其许可证兼容性
5. 定期审查和更新依赖项，特别是安全关键型依赖
6. 使用可选依赖组来分离不同环境的依赖需求（开发、测试、生产）
7. 避免直接修改锁文件，而是通过包管理器进行依赖更新

这种多语言依赖管理策略允许每个部分使用最适合的生态系统，同时保持整体项目的协调性。
