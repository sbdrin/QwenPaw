# 日志系统

QwenPaw 使用 Python 标准库的 `logging` 模块作为其日志记录的基础框架。整个项目的日志系统在 `src/qwenpaw/utils/logging.py` 中进行了集中配置和封装，提供了彩色控制台输出、文件输出以及跨平台兼容性等功能。

## 架构和约定

### 日志命名空间
- 所有日志记录都基于项目名称（`PROJECT_NAME`）定义的命名空间，即 `qwenpaw`
- 只有该命名空间下的日志才会被输出，避免了第三方依赖库的日志干扰

### 日志格式
- 控制台输出使用 `ColorFormatter`，带有颜色编码的日志级别标识
- 文件输出使用 `PlainFormatter`，包含时间戳、日志级别、文件路径和行号信息
- 两种格式器都会显示相对路径而非绝对路径

### 输出目标
- 控制台：使用 `StreamHandler` 输出到 `sys.stderr`，便于实时查看
- 文件：使用自定义的 `_SafeRotatingFileHandler` 进行文件输出，支持自动轮转（最大5MB，保留3个备份）

### 跨平台兼容性
- 在 Windows 上启用 ANSI 颜色支持
- 自定义的 `_SafeRotatingFileHandler` 处理 Windows 文件锁定错误，确保日志记录不会因文件锁定而中断

## 开发者应遵循的规则

### 获取日志记录器
```python
import logging

logger = logging.getLogger(__name__)
```

### 日志级别
- 使用标准的 Python 日志级别：`DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
- 根据信息的重要程度选择合适的日志级别

### 日志内容
- 日志消息应简洁明了，包含足够的上下文信息
- 避免在日志中记录敏感信息如密码、密钥等
- 使用结构化的日志消息以便后续分析

### 特殊处理
- 对于 `apscheduler` 等第三方库，系统会将其日志也写入项目主日志文件
- 提供了 `SuppressPathAccessLogFilter` 来过滤特定路径的访问日志

## 关键配置
- 日志文件路径：`WORKING_DIR / "qwenpaw.log"`
- 最大日志文件大小：5 MiB
- 备份文件数量：3个
- 日志轮转策略：按大小轮转
